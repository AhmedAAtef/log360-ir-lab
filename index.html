What is LOG360?

ManageEngine Log360 is a unified SIEM (Security Information and Event Management) solution built by Zoho Corporation. Unlike traditional standalone log collectors, LOG360 integrates multiple specialized security modules into a single dashboard—giving SOC analysts one place to monitor Active Directory, cloud environments, endpoints, web servers, firewalls, and databases across the entire enterprise infrastructure.

How LOG360 Works — Data Pipeline

LOG360 operates through a 6-stage security data pipeline that transforms raw logs into actionable SOC intelligence:

📥
1. Log Collection
Agent-based & agentless collection from 700+ sources
⚙️
2. Normalization
Parse, index & standardize raw logs (Elasticsearch engine)
🔗
3. Correlation
Connect related events across multiple log sources
🧠
4. UEBA Analytics
ML behavioral baselines detect anomalies & insider threats
🌐
5. Threat Intel
STIX/TAXII feeds match IoCs against known malicious IPs
🚨
6. Incident Response
SOAR playbooks trigger alerts, tickets & auto-containment

LOG360 Core Integrated Modules

🖥️
EventLog Analyzer
The primary SIEM engine. Centralizes log collection, normalization, archiving, and forensic analysis from 700+ sources (firewalls, servers, databases, switches, IDS/IPS). Supports Windows Event Logs, Syslog, and custom app logs.
🏢
ADAudit Plus
Real-time Active Directory auditing and monitoring. Tracks logon/logoff events (Event IDs 4624, 4625, 4634), GPO changes, user/group modifications, file integrity monitoring, and privilege escalation within Windows AD environments.
☁️
Cloud Security Plus
Provides visibility into public cloud infrastructure (AWS CloudTrail, Azure Activity Logs, GCP audit logs). Monitors cloud resource changes, IAM policy modifications, and storage bucket access patterns.
📧
M365 Security Plus
Audits Microsoft 365 environments including Exchange Online mailbox access, Azure AD/Entra ID sign-in logs, SharePoint permissions, Teams activity, and OneDrive file sharing events.
🛡️
UEBA & SOAR Add-ons
User and Entity Behavior Analytics (UEBA) uses machine learning to establish behavioral baselines and detect anomalies. Security Orchestration, Automation, and Response (SOAR) enables automated incident playbooks and ticket creation.

📖 LOG360 Search Query Syntax Reference

Before investigating the attack scenarios, familiarize yourself with the LOG360 search operators used in this lab. The search console below supports the following query syntax:

Operator / Technique Syntax Example Query Description
Keyword Search keyword powershell Search any field for the keyword
Filter by Log Source LogType:value ADAudit_Plus (use dropdown) Filter events from a specific LOG360 module
Filter by Event ID EventID 4625 Search for specific Windows Event ID numbers
Filter by Hostname hostname DC01.MEGACORP.LOCAL Isolate events from a specific server or endpoint
Filter by Username username a.davis Find all activity associated with a user account
Filter by Source IP IP address 10.14.22.99 Find all events originating from a specific IP
Filter by Severity CRITICAL / HIGH / INFO Use severity dropdown Isolate events by severity classification level
Process Name process_name vssadmin Search for specific process executions in Sysmon logs

🎯 Practice Questions — Test Your LOG360 Search Skills

Use the LOG360 SIEM Search Console below to answer these warm-up questions. These will help you become comfortable with filtering, searching, and navigating the log repository before tackling the real incident investigation.

Q1. How many total event logs are loaded in the LOG360 repository? (Look at the dataset counter above the search table)
Format: integer
Check the "Repository Loaded: X Total Logs" text next to the LOG360 Search Console title.
Q2. How many log events have CRITICAL severity? (Use the Severity dropdown filter)
Format: integer
Select "CRITICAL" from the severity dropdown and count the total results shown in the pagination bar.
Q3. What LOG360 module (Log Source) would you use to investigate Active Directory logon failures? (Use the Log Source dropdown)
Format: Module_Name
Which module name in the Log Source dropdown mentions "AD Events"? It starts with "ADAudit".
Q4. Search for Event ID 4625 in the LOG360 console. How many failed logon events appear in the results?
Format: integer
Type "4625" in the search box and press Search. The pagination bar shows "Showing X-Y of Z logs".
Q5. Filter by Log Source = Sysmon and Severity = CRITICAL. What hostname appears most frequently in the results?
Format: HOSTNAME.DOMAIN
Select "Sysmon" and "CRITICAL" from the dropdowns. Look at the Host column — which host appears in multiple CRITICAL Sysmon events?

Task 1 of 6 T1110.003 - Password Spraying

Task 1: Active Directory Password Spraying & Lockout

An attacker launched an NTLM password spray against Domain Controller DC01.MEGACORP.LOCAL.

1. What is the source IP address of the attacker initiating the password spray?
Format: X.X.X.X
Query LOG360 for ADAudit_Plus logs with Event ID 4625. Check the source_ip field.
2. Which domain user account was locked out (`Event ID 4740`) as a result?
Format: username
Filter for Event ID 4740 in ADAudit_Plus logs.
3. What sub_status hex code in Event ID 4625 logs indicates a bad password failure?
Format: 0xXXXXXXXX
Inspect the sub_status field in the failed logon event log.
✅ Task 1 Completed! LOG360 Investigation & Remediation Analysis
LOG360 Query: LogType:ADAudit_Plus AND (EventID:4625 OR EventID:4740)
Remediation Command: Disable-ADAccount -Identity "a.davis"
Task 2 of 6 T1059.001 - Encoded PowerShell

Task 2: Suspicious PowerShell Execution & Reverse Shell

An encoded PowerShell payload was spawned from a malicious phishing macro on WKSTN-FIN01.MEGACORP.LOCAL.

1. What destination port was targeted by the outbound PowerShell network connection?
Format: integer
Check Sysmon Event ID 3 for network connection destination_port.
2. What parent process launched `powershell.exe` on `WKSTN-FIN01.MEGACORP.LOCAL`?
Format: process.exe
Check Sysmon Event ID 1 parent_process field.
✅ Task 2 Completed! LOG360 Investigation & Remediation Analysis
LOG360 Query: LogType:Sysmon AND ProcessName:powershell.exe
Remediation Command: Stop-Process -Name "powershell" -Force
Task 3 of 6 T1543.003 - Service Persistence

Task 3: Privilege Escalation & Persistence via Service Creation

A backdoor service was installed in System log (`Event ID 7045`) for persistent administrative execution.

1. What is the exact service_name of the installed backdoor service?
Format: string
Search EventLog_Analyzer for Event ID 7045.
2. What is the full executable path specified for this malicious service?
Format: C:\path\to\file.exe
Examine service_file_name field in Event ID 7045 log.
✅ Task 3 Completed! LOG360 Investigation & Remediation Analysis
LOG360 Query: EventID:7045 AND Host:WKSTN-FIN01.MEGACORP.LOCAL
Remediation Command: sc.exe delete WinHealthMonitorSvc
Task 4 of 6 T1021.002 - PsExec Lateral Movement

Task 4: Lateral Movement via PsExec Pivoting

The attacker pivoted from WKSTN-FIN01 to file server FILE-SRV01 using stolen Domain Admin credentials.

1. What binary process name (`Sysmon Event ID 1`) was executed under `services.exe` during PsExec remote invocation?
Format: filename.exe
Check process_name for Sysmon Process Creation on FILE-SRV01.
✅ Task 4 Completed! LOG360 Investigation & Remediation Analysis
LOG360 Query: Host:FILE-SRV01.MEGACORP.LOCAL AND ProcessName:PSEXESVC.exe
Remediation Command: Set-ADAccountPassword -Identity "Administrator" -Reset
Task 5 of 6 T1505.003 - ASPX Web Shell

Task 5: Web Shell Execution on DMZ Web Server

An uploaded ASPX web shell script was executed on DMZ web server WEB-DMZ01. A network sensor also captured the interaction.

⬇️ Download capture.pcap
1. What is the exact URI path of the uploaded web shell script?
Format: /path/to/shell.aspx
Filter Web_Server_Log or Sysmon Event ID 11 on WEB-DMZ01.
2. What specific User-Agent string was used by the attacker to interact with the web shell?
Format: ToolName/version
Check user_agent in Web_Server_Log HTTP POST event.
3. Analyze the PCAP file in Wireshark. What exact command did the attacker send to the web shell?
Format: command string
Open capture.pcap in Wireshark, Right-Click the packet > Follow > TCP Stream. Look at the data at the bottom of the POST request.
✅ Task 5 Completed! LOG360 Investigation & Remediation Analysis
LOG360 Query: Host:WEB-DMZ01 AND LogType:Web_Server_Log
Remediation Command: Remove-Item "C:\inetpub\wwwroot\assets\vendors\shell_v2.aspx" -Force
Task 6 of 6 T1490 - Ransomware Shadow Copy Deletion

Task 6: Ransomware Execution & Shadow Copy Deletion

Volume shadow copies were wiped out prior to network share file encryption on FILE-SRV01.

1. What command flag argument was passed to `vssadmin.exe` to suppress confirmation prompts?
Format: /flag
Check command_line for vssadmin.exe in Sysmon logs.
2. What new file extension was appended to encrypted files on the file server?
Format: .EXTENSION
Look for share_name modifications in Event ID 5145 logs.
✅ Task 6 Completed! LOG360 Investigation & Remediation Analysis
LOG360 Query: ProcessName:vssadmin.exe OR EventID:5145
Remediation Action: Isolate Storage Server & Restore Immutable Snapshots
